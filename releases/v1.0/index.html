Purpose

This folder documents what belongs in the v1.0 release and how to upload/test it.

Files

customgpt-upload-manifest.md Upload strategy, required files, optional files, GPT instructions summary, and test cases. upload-checklist.md Pre-upload, Drive, GitHub, CustomGPT, compliance, and post-upload testing checklist.

Recommended Usage

  • Use this folder as the release control point for v1.0.
  • Do not upload release files to the GPT knowledge base unless you want the GPT to explain its own deployment package.
  • Use the checklist before updating CustomGPT knowledge files.

Upload rule: GitHub is the canonical source. Google Drive is the working/editing layer. CustomGPT receives approved release files only.
