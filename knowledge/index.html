Purpose

This folder contains the modular knowledge files that should be uploaded to the CustomGPT knowledge base.

Files

01-gpt-operating-instructions.md Core GPT behavior, purpose, guardrails, tone, intake, scenarios, and disclaimers. 02-minimum-forecast-intake-form.md Minimum intake fields needed before analyzing seller files. 03-ecommerce-cash-flow-forecasting-logic.md Forecast mechanics for 13-week ecommerce cash flow analysis. 04-messy-financial-data-handling-guide.md Rules for handling incomplete, stale, duplicated, or conflicting data. 05-forecast-output-templates.md Reusable report, dashboard, alert, memo, and action-plan templates. 06-ecommerce-risk-scoring-framework.md Risk scoring across cash, inventory, ads, debt, margin, funding readiness, and data quality. 07-funding-trigger-alert-framework.md Funding alert logic with strict no-approval/no-guarantee guardrails. 08-inventory-restock-recommendation-guide.md Inventory restock, stockout, overstock, and cash impact decision rules. 09-ecommerce-platform-data-guide.md How to interpret Shopify, Amazon, TikTok Shop, Etsy, WooCommerce, Stripe, PayPal, bank, ad, and P&L files. 10-plainspoken-operator-voice-guide.md Voice and tone guide for practical ecommerce operator language.

Recommended Usage

  • Upload the core files to the CustomGPT knowledge base in numerical order.
  • Keep these files focused on GPT behavior and analysis logic, not marketing copy.
  • Revise in GitHub first, then mirror approved updates to Drive and CustomGPT.

Upload rule: GitHub is the canonical source. Google Drive is the working/editing layer. CustomGPT receives approved release files only.
